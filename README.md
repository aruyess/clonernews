# ClonerNews

Лёгкий UI для [Hacker News API](https://github.com/HackerNews/API) на чистом HTML/CSS/JS — без фреймворков и сборки.

## Возможности

- **Ленты**: New, Top, Ask, Show, Jobs — переключаются в шапке
- **Типы постов**: stories, jobs, polls (с вариантами опроса и очками)
- **Комментарии**: вложенные, подгружаются по клику для каждого поста
- **Ленивая загрузка**: посты грузятся батчами по 20 при скролле (с debounce)
- **Live-обновления**: опрос `/maxitem` каждые 5 секунд, баннер при новых данных
- **Кэш запросов**: каждый элемент запрашивается не более одного раза за сессию
- **Сортировка**: от новых к старым (используется порядок `/newstories`)

## Запуск

Установка не требуется. Достаточно открыть `index.html` в браузере или поднять локальный сервер:

```bash
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

## Как это работает

| Задача | Где | Как |
|---|---|---|
| Получение ленты | `loadFeed()` | Запрос `/v0/<feed>.json` — массив id |
| Пагинация | `loadMore()` | Берёт срез `PAGE_SIZE=20` id, грузит элементы параллельно |
| Бесконечный скролл | `onScroll` + `debounce` | Вызывает `loadMore`, когда сентинел близко к viewport |
| Комментарии | `renderKids()` | Рекурсия по `item.kids`, раскрытие по клику |
| Опросы | `renderPollOptions()` | Обходит `item.parts`, грузит варианты ответов |
| Live-обновления | `pollLive()` + `setInterval` | Сравнивает `/maxitem` каждые 5 сек, показывает баннер обновления |
| Нагрузка на API | `cache` (Map) + `debounce` | Дедуп запросов, тротлинг обработчика скролла |

## API

Базовый URL: `https://hacker-news.firebaseio.com/v0/`

- `/item/<id>.json` — любой элемент (story / comment / job / poll / pollopt)
- `/newstories.json`, `/topstories.json`, `/askstories.json`, `/showstories.json`, `/jobstories.json` — списки id
- `/maxitem.json` — id последнего элемента (используется для live-обновлений)
- `/updates.json` — недавно изменённые элементы и профили

## Стратегия против rate-limiting

У HN API нет жёсткого лимита, но для бережного обращения:

- **In-memory Map-кэш** в `fetchItem` — каждый id запрашивается один раз
- **debounce(150ms)** на скролле — исключает шторм запросов при быстрой прокрутке
- **Пагинация по 20 элементов** вместо загрузки всех id сразу
- **Live-опрос раз в 5 секунд** — один запрос `maxitem` за тик
